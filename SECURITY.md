# Security Policy

If you find a security issue, please avoid opening a public GitHub issue with sensitive details.

Good examples include:

- leaked secrets or tokens
- account or session exposure
- unsafe order placement behavior
- ways to bypass confirmation gates
- vulnerabilities in optional integrations

## How to report

Open a private security advisory on GitHub for this repository, or contact the maintainer directly through a private channel if one is listed on the profile.

Please include:

- a short description of the issue
- steps to reproduce it
- impact
- any suggested mitigation

I’ll aim to acknowledge real reports quickly and fix or mitigate them as soon as I can.
